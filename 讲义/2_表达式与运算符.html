表达式与运算符

掌握C语言中各种运算的规则，让数据"动起来"

表达式基础

在第1章中，我们学习了如何用变量来存储数据。但光有数据还不够，我们还需要对数据进行各种操作——就像有了食材还需要烹饪方法一样。运算符就是对数据进行操作的"厨具"，而表达式就是用运算符把数据组合起来的"菜谱"。

新手注意：
  • 表达式就像数学中的算式，由操作数（数据）和运算符（操作符号）组成
  • 每个表达式都会计算出一个确定的结果
  • 运算符的优先级决定了计算的先后顺序
  • C语言中 = 是赋值，== 才是判断相等，不要混淆！

表达式的组成

一个表达式可以很简单，也可以很复杂：

  • 最简单的表达式：一个常量或变量，如 42、score
  • 常见的表达式：用运算符连接操作数，如 a + b、x * 2 + 1
  • 复杂的表达式：嵌套多个运算符，如 (a + b) * (c - d) / e % f

零基础学习路线

如果你刚接触C语言，可以按以下节奏循序渐进：

  1. 认得符号 —— 先弄清每个运算符的读法和含义，把它当成"数学符号升级版"
  2. 会读表达式 —— 像读中文一样从左到右读出含义，在草稿上写出计算顺序
  3. 亲手敲代码 —— 复制示例后，再尝试改动数字或运算符，观察结果的变化
  4. 解释给别人听 —— 能把表达式的执行过程讲出来，就说明你真的懂了

任何一个环节不明白都不要紧，回到上一步重复练习，保持"慢慢来、反复看、动手做"的节奏即可。

生活类比
  • 把表达式想成做蛋糕的配方：材料（变量）+ 操作（运算符）
  • 先准备好材料，再按步骤混合，否则味道就会怪怪的
检查清单
  • 每个变量是否有初始值？
  • 是否记得加括号确保顺序？
  • 是否考虑到了整数/浮点数的差别？
坚持练习
  • 每天随手写3个表达式并预测结果
  • 用printf打印预测值与真实值对比
  • 把看不懂的地方记在"疑问本"里

算术运算符

五种基本算术运算符

算术运算符是我们最熟悉的运算符，和数学中的加减乘除几乎一样——不过C语言多了一个"取模"运算符。

运算符 说明 示例 结果
+ 加法 5 + 3 8
- 减法 10 - 4 6
* 乘法 7 * 6 42
/ 除法 15 / 3 5
% 取模（求余数） 17 % 5 2

整数除法——最容易踩的坑

在C语言中，两个整数相除，结果还是整数，小数部分会被直接丢弃（不是四舍五入！）。这是初学者最容易犯错的地方。

生活类比：分糖果
  • 10块糖分给3个小朋友 → 每人3块（余1块，余的不管）
  • 7块糖分给2个小朋友 → 每人3块（余1块，余的不管）
  • 结果总是整数，小数部分直接丢弃
整数除法 vs 浮点除法
#include <stdio.h>

int main() {
    int a = 10, b = 3;

    // 整数除法：结果向下取整，小数部分丢弃
    printf("整数除法: %d / %d = %d\n", a, b, a / b);  // 输出: 3

    // 想得到小数结果？至少有一个操作数是浮点数
    printf("浮点除法: %.1f / %d = %.1f\n", 10.0, b, 10.0 / b);  // 输出: 3.3

    return 0;
}
常见错误：

计算平均值时，int / int 会丢失精度！

  • (3 + 4) / 2 的结果是 3，不是 3.5
  • 正确做法：(3.0 + 4) / 2 或 (float)(3 + 4) / 2

取模运算符 %——求余数的利器

取模运算符 % 用于计算两个整数相除后的余数。它在编程中非常有用！

取模运算的常见用途：
  • 判断奇偶：n % 2 == 0 为偶数，n % 2 != 0 为奇数
  • 获取个位数：123 % 10 得到 3
  • 循环计数：i % 7 可以让值在0~6之间循环
  • 时间换算：125秒 / 60 = 2分，125秒 % 60 = 5秒
取模运算示例
#include <stdio.h>

int main() {
    int seconds = 125;

    // 125秒 = 2分5秒
    printf("%d秒 = %d分%d秒\n", seconds, seconds / 60, seconds % 60);

    // 判断奇偶
    int num = 7;
    printf("%d是%s数\n", num, num % 2 == 0 ? "偶" : "奇");

    return 0;
}
注意：
  • % 只能用于整数，不能用于浮点数！5.0 % 2 是错误的
  • 负数取模的结果符号取决于编译器实现，建议避免对负数取模

一元负号运算符

- 除了做减法，还可以作为一元运算符对数字取负：

一元负号示例
int a = 5;
int b = -a;  // b = -5
int c = -(-a);  // c = 5，负负得正

分离三位数的各位

关系与逻辑运算符

关系运算符——做判断的基础

关系运算符用于比较两个值的大小关系，结果是"真"（1）或"假"（0）。就像老师批改试卷，判断对还是错。

运算符 说明 示例 结果
> 大于 5 > 3 1（真）
< 小于 5 < 3 0（假）
>= 大于等于 5 >= 5 1（真）
<= 小于等于 5 <= 4 0（假）
== 等于 5 == 5 1（真）
!= 不等于 5 != 3 1（真）
最常见的错误：

把 ==（判断相等）写成 =（赋值）！

  • if (a = 5) —— 这是把5赋值给a，永远为真！
  • if (a == 5) —— 这才是判断a是否等于5

逻辑运算符——组合判断条件

逻辑运算符用于将多个条件组合在一起，就像生活中的"并且"和"或者"：

  • "考试及格并且出勤达标" → 用 &&（逻辑与）
  • "周六或者周日" → 用 ||（逻辑或）
  • "不是会员" → 用 !（逻辑非）
运算符 说明 示例 含义
&& 逻辑与（AND） a > 0 && b > 0 a大于0且b大于0
|| 逻辑或（OR） a > 0 || b > 0 a大于0或b大于0
! 逻辑非（NOT） !(a > 0) a不大于0

逻辑运算真值表

C语言中，0为假，非0为真。逻辑运算的结果只有0或1：

逻辑与 &&
  • 真 && 真 = 真
  • 真 && 假 = 假
  • 假 && 真 = 假
  • 假 && 假 = 假
逻辑或 ||
  • 真 || 真 = 真
  • 真 || 假 = 真
  • 假 || 真 = 真
  • 假 || 假 = 假
逻辑非 !
  • !真 = 假
  • !假 = 真
  • !0 = 1
  • !5 = 0（非0即真）

关系与逻辑运算符综合示例

综合使用示例
#include <stdio.h>

int main() {
    int score = 85;

    // 判断成绩是否在优秀范围
    printf("是否优秀: %d\n", score >= 90);  // 输出: 0（假）

    // 判断成绩是否及格
    printf("是否及格: %d\n", score >= 60);  // 输出: 1（真）

    // 组合判断：是否在80~90之间
    printf("是否良好: %d\n", score >= 80 && score < 90);  // 输出: 1（真）

    // 逻辑非：是否不及格
    printf("是否不及格: %d\n", !(score >= 60));  // 输出: 0（假）

    return 0;
}

逻辑运算符的优先级

在逻辑运算符之间，优先级从高到低为：! > && > ||。建议在复杂表达式中使用括号明确意图。

逻辑运算符优先级示例
// !a || b && !c 等价于 (!a) || (b && (!c))
int a = 1, b = 2, c = 3;
int result = !a || b && !c;

// 分析过程:
// !a = !1 = 0
// !c = !3 = 0
// b && !c = 2 && 0 = 0
// 0 || 0 = 0
printf("result = %d\n", result);  // 输出: 0

赋值运算符

基本赋值与复合赋值

赋值运算符 = 的作用是"把右边的值放到左边的变量中"。C语言还提供了复合赋值运算符，让代码更简洁。

重要概念：

C语言中的 = 不是数学中的"等于"，而是"赋值"——就像把水倒进杯子里，把右边的值"倒进"左边的变量。

运算符 说明 示例 等价形式
= 赋值 a = 5 —
+= 加法赋值 a += 3 a = a + 3
-= 减法赋值 a -= 2 a = a - 2
*= 乘法赋值 a *= 4 a = a * 4
/= 除法赋值 a /= 2 a = a / 2
%= 取模赋值 a %= 3 a = a % 3

复合赋值的陷阱

复合赋值运算符的右边表达式会被当作一个整体来计算，这和直觉可能不同！

注意：

a *= b + 5 等价于 a = a * (b + 5)，不是 a = a * b + 5！

复合赋值陷阱示例
#include <stdio.h>

int main() {
    int x = 2, y = 3;

    // x *= y + 5 等价于 x = x * (y + 5)
    x *= y + 5;
    printf("x = %d\n", x);  // 输出: 16 (因为 2 * (3+5) = 16)
    // 如果是 x = x * y + 5，结果应该是 2*3+5 = 11

    return 0;
}

连续赋值

赋值运算符的结合性是从右到左，所以可以连续赋值：

连续赋值示例
#include <stdio.h>

int main() {
    int a, b, c;

    // 连续赋值：从右到左执行
    a = b = c = 5;

    // 执行过程: c = 5 → b = 5 → a = 5
    printf("a=%d, b=%d, c=%d\n", a, b, c);  // 输出: a=5, b=5, c=5

    return 0;
}

自增自减运算符

前缀与后缀的区别

自增 ++ 和自减 -- 是C语言的特色运算符，让变量自身加1或减1。关键要搞清楚前缀和后缀的区别。

生活类比：

前缀自增就像"先吃饭再付钱"，后缀自增就像"先付钱再吃饭"：

  • ++i：先把i加1，再用i的新值
  • i++：先用i的旧值，再把i加1
运算符 名称 含义 示例（设 i=5）
++i 前缀自增 先加1，再使用 表达式的值为6，i变为6
i++ 后缀自增 先使用，再加1 表达式的值为5，i变为6
--i 前缀自减 先减1，再使用 表达式的值为4，i变为4
i-- 后缀自减 先使用，再减1 表达式的值为5，i变为4

代码示例

前缀与后缀对比
#include <stdio.h>

int main() {
    int x = 5, y, z;

    // 前缀自增：先增加x，再赋值给y
    y = ++x;  // x先变为6，然后y=6
    printf("前缀: y = %d, x = %d\n", y, x);  // 输出: y=6, x=6

    // 重置x
    x = 5;

    // 后缀自增：先赋值给z，再增加x
    z = x++;  // z=5（x的旧值），然后x变为6
    printf("后缀: z = %d, x = %d\n", z, x);  // 输出: z=5, x=6

    return 0;
}

单独使用时没区别

如果自增/自减是单独一条语句（不在更大的表达式中），前缀和后缀效果完全相同：

单独使用
int i = 5;
i++;   // i变为6
++i;   // i变为7
// 两者效果完全一样，推荐用 i++ 更常见
新手建议：

在复杂表达式中避免使用自增自减，容易出错。尽量单独使用，让代码更清晰。

边学边练 - 自增自减

代码

三元运算符

条件运算符 ? :

三元运算符是C语言中唯一需要三个操作数的运算符，可以把它当作简化版的 if-else：

语法格式
条件表达式 ? 表达式1 : 表达式2

// 如果条件为真，返回表达式1的值
// 如果条件为假，返回表达式2的值
生活类比：

就像餐厅的"满减规则"——消费满100元？打8折：原价。

  • 条件 ? 满足时 : 不满足时

三元运算符示例

常见用法
#include <stdio.h>

int main() {
    int a = 10, b = 20;

    // 求两个数中的较大值
    int max = (a > b) ? a : b;
    printf("较大值: %d\n", max);  // 输出: 20

    // 判断奇偶
    printf("%d是%s数\n", a, a % 2 == 0 ? "偶" : "奇");  // 输出: 10是偶数

    // 嵌套使用：给成绩分级
    int score = 75;
    printf("成绩: %s\n", score >= 90 ? "优秀" : score >= 60 ? "及格" : "不及格");  // 输出: 及格

    return 0;
}
使用建议：
  • 简单的二选一用三元运算符很方便
  • 嵌套层次太多时，还是用 if-else 更清晰
  • 三元运算符的优先级很低，复杂表达式建议加括号

运算符优先级与结合性

为什么要了解优先级？

就像数学中"先乘除后加减"一样，C语言的运算符也有固定的计算顺序。搞不清优先级，表达式可能完全不是你以为的意思！

实用原则：
  • 记不住优先级？加括号！括号永远是最清晰的
  • 只需要记住：算术 > 关系 > 逻辑 > 赋值
  • 在逻辑运算符中：! > && > ||

完整优先级表（从高到低）

优先级 运算符 结合性 说明
1（最高） () [] -> . 左到右 括号、数组下标、成员访问
2 ! ++ -- -(负号) (类型) sizeof 右到左 逻辑非、自增自减、负号、强制转换
3 * / % 左到右 乘法、除法、取模
4 + - 左到右 加法、减法
5 < <= > >= 左到右 关系运算符
6 == != 左到右 相等性运算符
7 && 左到右 逻辑与
8 || 左到右 逻辑或
9 ?: 右到左 三元运算符
10（最低） = += -= *= /= %= 右到左 赋值运算符
注：本表仅列出本章涉及的运算符，完整的优先级表还包含位运算符等，将在后续章节介绍。

优先级实战分析

复杂表达式求值
#include <stdio.h>

int main() {
    int a = 1, b = 2, c = 3;

    // 示例1：算术运算优先级
    int r1 = a + b * c;       // 先算 b*c=6，再算 a+6=7
    printf("a + b * c = %d\n", r1);  // 输出: 7

    // 示例2：关系与逻辑混合
    int r2 = a + b > c && c != 0;
    // 分析: (a+b) > c → 3 > 3 → 0
    //       c != 0 → 1
    //       0 && 1 → 0
    printf("结果 = %d\n", r2);  // 输出: 0

    return 0;
}

运算中的类型提升

知识点回顾：

第1章已介绍了类型转换的基本概念（自动类型转换和强制类型转换）。本章聚焦于表达式求值过程中的类型提升规则，帮助你理解运算符如何处理不同类型的操作数。

整型提升与寻常算术转换

在表达式中，C语言会按照两步规则自动提升操作数的类型：

  • 第1步——整型提升：char 和 short 在运算前自动提升为 int
  • 第2步——寻常算术转换：如果两个操作数类型不同，"较小"的类型自动提升为"较大"的类型
char / short → int → unsigned int → long → double
float → double
记忆口诀：

"小转大，不丢精度"——整数变浮点数没问题，但浮点数变整数会截断小数部分。

运算中的类型提升示例

类型提升在表达式中的体现
#include <stdio.h>

int main() {
    // 示例1：char参与运算时自动提升为int
    char ch = 'A';        // ASCII值为65
    int result1 = ch + 1;   // 65 + 1 = 66，即'B'
    printf("'A' + 1 = %c(%d)\n", result1, result1);

    // 示例2：int和double运算，int提升为double
    int a = 5;
    double b = 2.0;
    printf("5 / 2.0 = %.1f\n", a / b);  // 5.0/2.0 = 2.5

    // 示例3：强制转换改变运算结果
    printf("5 / 2 = %d\n", 5 / 2);              // 整数除法: 2
    printf("(double)5 / 2 = %.1f\n", (double)5 / 2);  // 浮点除法: 2.5

    // 示例4：强制截断小数
    double result2 = (int)3.14 + 1.8;
    printf("(int)3.14 + 1.8 = %.1f\n", result2);  // 3 + 1.8 = 4.8

    return 0;
}

sizeof运算符

sizeof 运算符用于获取数据类型或变量占用的字节数，可以帮助我们验证类型提升的效果：

sizeof使用示例
#include <stdio.h>

int main() {
    // 不同类型的字节大小
    printf("sizeof(int)    = %u\n", sizeof(int));      // 4
    printf("sizeof(double) = %u\n", sizeof(double));   // 8
    printf("sizeof(char)   = %u\n", sizeof(char));     // 1

    // 表达式的类型由操作数决定
    printf("sizeof(5/2)     = %u\n", sizeof(5/2));      // 4 (int)
    printf("sizeof(5.0/2)   = %u\n", sizeof(5.0/2));    // 8 (double)

    return 0;
}

短路求值与未定义行为

什么是短路求值？

C语言的逻辑运算符有一个重要特性：只要能确定结果，就不再继续计算。这就像短路一样，电流找到了捷径，就不走原来的路了。

  • && 短路：如果第一个条件为假，结果必定为假，第二个条件不再计算
  • || 短路：如果第一个条件为真，结果必定为真，第二个条件不再计算
生活类比：
  • 逻辑与就像"必须所有门都打开才能通过"——第一扇门就锁着，后面的门不用看了
  • 逻辑或就像"任意一扇门打开就能通过"——第一扇门开着，后面的门不用看了

短路求值的实用场景

前置提示：

下面的示例用到了 if-else 语句，它将在第3章详细介绍。这里只需关注逻辑运算符的短路效果即可。

短路求值实例
#include <stdio.h>

int main() {
    // 场景1：避免除零错误
    int x = 0;
    // 如果 x != 0 为假，就不会执行 10/x，避免除零崩溃
    if (x != 0 && 10 / x > 5) {
        printf("条件成立\n");
    } else {
        printf("条件不成立\n");  // 输出这个
    }

    // 场景2：指针保护（后续章节会学）
    // if (ptr != NULL && *ptr > 0) { ... }
    // 如果指针为空，就不会去解引用，避免崩溃

    // 场景3：注意自增在短路中的行为
    int a = 5, b = 0;
    int result = (b != 0) && (a++ > 0);
    printf("a = %d\n", a);  // 输出: 5（短路了，a++没有执行！）

    return 0;
}
重要提醒：

不要在逻辑运算的右侧放置有副作用的表达式（如 i++、赋值操作），因为短路可能导致它们不被执行，结果难以预料！

未定义行为——程序员的噩梦

未定义行为（Undefined Behavior, UB）是指C语言标准没有规定结果的行为。遇到未定义行为，程序可能崩溃、可能输出乱码、也可能看起来"正常"运行——但结果完全不可靠。

未定义行为的危险：
  • 在你的电脑上运行正常，换一台电脑就可能出错
  • 今天运行正常，明天可能就崩溃
  • Debug模式正常，Release模式出错
  • 编译器不同，结果也不同

常见的未定义行为

未定义行为示例
#include <stdio.h>

int main() {
    int i = 5;

    //  未定义行为1：同一表达式中对同一变量多次修改
    // int j = i++ + ++i;  // i被修改了两次，结果不可预测！
    // 有的编译器算出11，有的算出12，都是"合法"的

    //  正确做法：分开操作
    int j1 = i++;  // j1 = 5, i = 6
    int j2 = ++i;  // j2 = 7, i = 7
    int sum = j1 + j2;
    printf("sum = %d\n", sum);  // 输出: 12

    //  未定义行为2：多个序列点之间修改同一变量
    // int k = (i = 3) + (i = 5);  // i被赋值两次，未定义！

    //  未定义行为3：除以零
    // int z = 10 / 0;  // 程序直接崩溃！

    return 0;
}
如何避免未定义行为：
  • 不要在同一个表达式中对同一个变量多次使用自增/自减
  • 除法运算前检查除数是否为0
  • 不确定时，把复杂表达式拆成多行简单语句
  • 编译时开启警告：gcc -Wall -Wextra，让编译器帮你找问题
边学边练 - 短路求值与未定义行为

练习4： 以下代码是否有未定义行为？如果有，请指出并修正：

代码
int i = 3;
int a = i++ + i++;
参考答案

是的，这是未定义行为！

在同一个表达式中，i 被自增了两次，C语言标准没有规定哪个自增先执行，不同编译器可能给出不同结果。

修正方法：

修正后代码
int i = 3;
int a = i++;  // a = 3, i = 4
a += i++;      // a = 3 + 4 = 7, i = 5

逗号与sizeof运算符

逗号运算符

逗号运算符 , 是C语言中优先级最低的运算符，用于将多个表达式组合为一个表达式。逗号表达式从左到右依次求值，整个表达式的值是最右边表达式的值。

逗号运算符示例
#include <stdio.h>

int main() {
    // 逗号表达式：从左到右求值，结果是最后一个
    int result = (1, 2, 3);
    printf("result = %d\n", result);  // 输出: result = 3

    // 常见用法：for循环中同时操作多个变量
    int sum = 0, i;
    for (i = 1; i <= 10; i++, sum += i)
        ;  // 空循环体，逗号表达式在更新部分同时执行两个操作
    printf("sum = %d\n", sum);

    return 0;
}
注意区分：
  • 函数参数中的逗号是分隔符，不是逗号运算符：printf("%d %d", a, b)
  • 变量声明中的逗号是分隔符：int a = 1, b = 2;
  • 只有当逗号在表达式中作为运算符使用时，才从左到右求值并返回最后一个值

sizeof运算符

sizeof 是一个编译时运算符，用于获取数据类型或变量占用的内存字节数。它在第1章已经简单介绍过，这里做更详细的说明。

sizeof 各种用法
#include <stdio.h>

int main() {
    // 对类型使用sizeof（必须加括号）
    printf("int:    %zu 字节\n", sizeof(int));
    printf("double: %zu 字节\n", sizeof(double));

    // 对变量使用sizeof（括号可省略，但建议保留）
    int age = 20;
    printf("age变量: %zu 字节\n", sizeof age);

    // 对表达式使用sizeof（不计算表达式，只看结果类型）
    printf("3+5.0: %zu 字节\n", sizeof(3 + 5.0));  // double: 8字节

    return 0;
}
sizeof的特点：
  • sizeof在编译时就确定了结果，不会在运行时计算
  • sizeof的返回类型是 size_t（无符号整数），用 %zu 格式输出
  • sizeof对数组的使用将在第5章详细介绍

位运算初步

什么是位运算？

之前我们学习的运算符都是以整个数值为单位进行操作的。而位运算符直接对整数的二进制位进行操作。位运算在第11章会详细讲解，这里先做初步了解。

前置知识回顾：

在第0章我们学习了二进制。例如，整数5在二进制中表示为 0101，整数3表示为 0011。位运算就是直接操作这些0和1。

位运算符一览

运算符 名称 功能 示例（5 op 3）
&按位与两位都为1才为15 & 3 = 1
|按位或有一位为1就为15 | 3 = 7
^按位异或相同为0，不同为15 ^ 3 = 6
~按位取反0变1，1变0~5 = -6
<<左移所有位左移n位5 << 1 = 10
>>右移所有位右移n位5 >> 1 = 2
位运算基本示例
#include <stdio.h>

int main() {
    unsigned int a = 5;  // 二进制: 0101
    unsigned int b = 3;  // 二进制: 0011

    printf("a & b = %d\n", a & b);   // 0001 = 1
    printf("a | b = %d\n", a | b);   // 0111 = 7
    printf("a ^ b = %d\n", a ^ b);   // 0110 = 6
    printf("a << 1 = %d\n", a << 1); // 1010 = 10
    printf("a >> 1 = %d\n", a >> 1); // 0010 = 2

    return 0;
}
小技巧：
  • 左移1位相当于乘以2：n << 1 等价于 n * 2
  • 右移1位相当于除以2：n >> 1 等价于 n / 2（整数除法）
  • 位运算的执行速度比乘除法快，但现代编译器通常会自动优化
边学边练 - 位运算初步

练习1： 计算 12 & 10 的结果（写出二进制计算过程）。

参考答案 
  12 = 1100
  10 = 1010
  ---------
  &   = 1000 = 8

所以 12 & 10 = 8

练习2： 如果 int x = 1;，那么 x << 3 的结果是多少？

参考答案

x << 3 将1左移3位，相当于 1 * 2³ = 8。二进制过程：0001 → 1000 = 8。

本章小结

重点回顾

  • 算术运算符：掌握 + - * / % 五种运算，特别注意整数除法截断和取模运算的用途
  • 关系与逻辑运算符：关系运算结果是0或1，逻辑运算符用于组合条件，== 和 = 不要混淆
  • 赋值运算符：= 是赋值不是等于，复合赋值的右边是整体
  • 自增自减：前缀"先变后用"，后缀"先用后变"，单独使用时无区别
  • 三元运算符：简化版的 if-else，适合简单的二选一
  • 运算符优先级：算术 > 关系 > 逻辑 > 赋值，拿不准就加括号
  • 类型转换：隐式转换"小转大"，显式转换用 (类型)
  • 短路求值：&& 遇假即停，|| 遇真即停，可用来避免错误
  • 未定义行为：同一表达式中对同一变量多次修改是UB，必须避免
  • 逗号运算符：从左到右求值，结果为最后一个表达式
  • sizeof运算符：编译时确定类型大小，返回size_t
  • 位运算初步：直接操作二进制位，左移相当于乘2，右移相当于除2
