为什么要学习C语言？

  • 应用广泛：操作系统、嵌入式系统、数据库等都使用C语言开发
  • 高效强大：C语言可以直接操作硬件，执行效率极高
  • 学习基础：掌握C语言后，学习C++、Java、Python等语言会更加容易

计算机基础概念

计算机是如何工作的？

计算机本质上是一个能够执行指令的电子设备。就像一个严格按照菜谱做菜的厨师——你给它指令（菜谱），它就一步一步地执行。

计算机的五大组成部分

运算器（ALU）
  • 负责执行算术运算（加减乘除）
  • 负责执行逻辑运算（与或非）
  • 相当于厨师的"手"
控制器
  • 指挥各部件协调工作
  • 按程序指令顺序执行
  • 相当于厨师的"大脑"
存储器（内存）
  • 存储程序和数据
  • 每个字节有唯一地址
  • 相当于厨师的"工作台"
输入设备
  • 键盘、鼠标、扫描仪
  • 向计算机输入数据
  • 相当于"采购的食材"
输出设备
  • 显示器、打印机
  • 显示处理结果
  • 相当于"端上桌的菜"

内存的概念

内存是计算机中最重要的概念之一，理解内存对学习C语言至关重要：

  • 内存由许多字节（Byte）组成，每个字节有唯一的地址
  • 每个字节可以存储一个0-255之间的数字
  • 多个字节可以组合存储更大的数据（如 int 占用4个字节）
  • 程序运行时，变量就存储在内存中
形象比喻：

想象内存是一排带编号的小盒子（地址），每个盒子可以存放一张写有数字的纸条。程序通过盒子编号（地址）来存取数据。

程序是如何运行的？

从编写代码到程序运行，就像把中文翻译成外语再让外国人阅读一样，需要经历以下步骤：

1. 编写源代码 → 保存为 .c 文件
2. 编译（Compile） → 翻译成二进制机器码
3. 链接（Link） → 连接系统库，生成 .exe
4. 运行（Run） → 加载到内存，CPU执行
详细说明：
  • 编写源代码：程序员使用编程语言（如C语言）编写代码，保存为文本文件
  • 编译：编译器将人类可读的源代码翻译成计算机可执行的二进制机器码
  • 链接：链接器将编译后的代码与系统库文件连接，生成可执行文件
  • 运行：操作系统加载可执行文件到内存，CPU开始执行程序指令
重要提示：

编译错误和运行时错误是不同的：

  • 编译错误（Compile Error）：代码语法有问题，程序无法生成。需要修改代码后重新编译。
  • 运行时错误（Runtime Error）：程序语法正确，但执行时出现问题（如除以零）。

二进制与数据表示

什么是二进制？

计算机内部使用二进制（Binary）来表示所有数据。二进制只使用两个数字：0和1。就像电灯开关只有"开"和"关"两种状态一样。

为什么使用二进制？

  • 电子元件最容易表示两种状态：通电（1）和断电（0）
  • 二进制运算规则简单，易于硬件实现
  • 可靠性高，抗干扰能力强

二进制与十进制的转换

十进制转二进制（除2取余法）：

以十进制数13为例：

十进制转二进制
13 ÷ 2 = 6 余 1
 6 ÷ 2 = 3 余 0
 3 ÷ 2 = 1 余 1
 1 ÷ 2 = 0 余 1

从下往上读：1101
所以 13（十进制）= 1101（二进制）
二进制转十进制（按权展开）：

以二进制数1101为例：

二进制转十进制
1×2³ + 1×2² + 0×2¹ + 1×2⁰
= 8 + 4 + 0 + 1
= 13

单位 大小 说明
位（bit） 0或1 最小的数据单位
字节（Byte） 8位 基本存储单位，可表示0-255
KB（千字节） 1024字节 约1000字节
MB（兆字节） 1024 KB 约100万字节

什么是编程

编程的本质

编程（Programming）就是编写计算机程序的过程。简单来说，就是告诉计算机：

  • 做什么（What to do）
  • 怎么做（How to do it）

编程语言的作用

计算机只能理解机器语言（二进制代码），但人类很难直接用机器语言编程。因此，人们发明了编程语言作为人与计算机之间的桥梁——就像翻译官帮你和外国人沟通一样。

编程语言的层次：
  • 机器语言：计算机直接执行的二进制代码（010101...）
  • 汇编语言：用助记符表示机器指令（如 MOV, ADD）
  • 高级语言：接近人类语言（如 C、Python、Java）

C语言的特点

结构化
  • 程序由函数组成
  • 结构清晰，易于理解
高效
  • 生成的代码执行效率高
  • 接近硬件层面操作
灵活
  • 可以直接操作内存
  • 可以控制硬件
可移植
  • 程序可以在不同平台上编译运行
  • "一次编写，处处编译"

算法与流程图

算法（Algorithm）是解决问题的一系列步骤。就像做菜的菜谱，每一步都有明确的操作。在编程之前，通常需要先设计算法。

算法的特征

  • 有穷性：算法必须在有限步骤内结束
  • 确定性：每一步都有明确的含义
  • 可行性：每一步都可以被执行
  • 输入：有零个或多个输入
  • 输出：有一个或多个输出
示例：计算两个数的和

算法步骤：

  1. 接收两个数作为输入
  2. 将这两个数相加
  3. 输出结果

VC++2010使用教程

安装Visual C++ 2010

系统要求

  • 操作系统：Windows XP / Vista / 7 / 8 / 10
  • 内存：至少1GB RAM（推荐2GB）
  • 硬盘空间：至少3GB可用空间
安装流程：
  1. 下载安装程序：获取Visual C++ 2010 Express安装程序（免费版本）
  2. 运行安装程序：双击安装程序，按照向导提示进行安装
  3. 选择安装组件：选择"Visual C++"组件进行安装
  4. 完成安装：等待安装完成，点击"完成"按钮

创建第一个C语言项目

创建项目流程：
  1. 启动VC++2010：点击"开始"菜单 → "所有程序" → "Microsoft Visual Studio 2010" → "Microsoft Visual C++ 2010 Express"
  2. 新建项目：点击菜单"文件" → "新建" → "项目"（或按 Ctrl+Shift+N）
  3. 选择项目类型：
    • 左侧选择"Visual C++" → "Win32"
    • 中间选择"Win32控制台应用程序"
    • 输入项目名称（如"HelloWorld"）
    • 选择保存位置
    • 点击"确定"
  4. 应用程序向导：点击"下一步"，在"应用程序设置"中勾选"空项目"，点击"完成"
  5. 添加源文件：
    • 在"解决方案资源管理器"中右键点击"源文件"文件夹
    • 选择"添加" → "新建项"
    • 选择"C++文件(.cpp)"
    • 输入文件名（如"main.c"，注意改为 .c 扩展名）
    • 点击"添加"
重要提示：

虽然VC++2010是C++编译器，但它完全兼容C语言。将文件保存为 .c 扩展名，编译器会按C语言规则编译。

编译和运行程序

两种编译方式：
  • 方法1：点击"生成" → "生成解决方案"（或按 F7）
  • 方法2：点击工具栏上的"生成"按钮
两种运行方式：
  • 方法1：点击"调试" → "开始执行（不调试）"（或按 Ctrl+F5）
  • 方法2：点击工具栏上的"开始执行（不调试）"按钮
注意：

如果直接按 F5（开始调试），程序运行后会立即关闭，看不到输出结果。建议初学者使用 Ctrl+F5 运行程序，这样程序执行完毕后会显示"按任意键继续..."。

调试技巧

调试（Debugging）是查找和修复程序错误的过程，就像侦探破案一样，一步一步找到问题所在。

四大调试技巧：
  • 1. 设置断点：在代码行号左侧点击，出现红点，表示在此处暂停执行
  • 2. 单步执行：F11（逐语句，进入函数内部）；F10（逐过程，不进入函数内部）
  • 3. 查看变量：在调试过程中，将鼠标悬停在变量上可查看当前值
  • 4. 监视窗口：点击"调试" → "窗口" → "监视"，可以持续监视变量的值

第一个C语言程序

Hello World程序

按照传统，学习一门新编程语言的第一个程序是输出"Hello, World!"。就像学书法先写"永"字一样，这是每个程序员的第一步。

Hello World
/*
 * 第一个C语言程序
 * 功能：在屏幕上显示 "Hello, World!"
 */

#include <stdio.h>

int main() {
    printf("Hello, World!\n");
    return 0;
}

代码逐行解释

逐行解读：
  • 第1-4行：注释 — /* ... */ 是多行注释，用于说明代码的功能，编译器会忽略注释内容
  • 第6行：包含头文件 — #include <stdio.h> 告诉编译器包含标准输入输出库，这样我们才能使用 printf 函数
  • 第8行：main函数 — int main() 是程序的入口点，每个C程序都必须有且只有一个main函数
  • 第9行：printf语句 — printf() 是一个输出函数，用于在屏幕上显示文本。\n 表示换行
  • 第10行：return语句 — return 0; 表示程序正常结束，向操作系统返回0

C语言程序的基本结构

程序的基本组成部分

程序结构示例
/*
 * 程序说明：计算两个数的和
 * 作者：张三
 * 日期：2024年
 */

// 第1部分：预处理指令（头文件包含）
#include <stdio.h>

// 第2部分：全局声明（可选）
int global_var = 10;  // 全局变量

// 第3部分：函数声明（如果有自定义函数）
int add(int a, int b);

// 第4部分：main函数（程序入口）
int main() {
    // 局部变量声明
    int num1, num2, sum;

    // 输入
    printf("请输入两个整数：");
    scanf("%d %d", &num1, &num2);

    // 处理
    sum = add(num1, num2);

    // 输出
    printf("两数之和为：%d\n", sum);

    return 0;
}

// 第5部分：函数定义
int add(int a, int b) {
    return a + b;
}

各部分说明

五个组成部分：
  • 1. 注释：用于说明代码功能，提高可读性。可以是单行注释（//）或多行注释（/* */）
  • 2. 预处理指令：以 # 开头，在编译前执行。最常用的是 #include 用于包含头文件
  • 3. 全局声明：在所有函数之外声明的变量和函数，整个程序都可以使用
  • 4. main函数：程序的入口点，必须有且只有一个。程序从这里开始执行
  • 5. 自定义函数：将功能模块化，提高代码的复用性和可读性

编码规范

良好的编码习惯可以让代码更易读、易维护，就像写文章要有段落和标点一样。

命名规范：
  • 变量名使用小写字母，多个单词用下划线连接（如：student_name）
  • 常量名使用大写字母（如：MAX_SIZE）
  • 函数名使用小写字母，多个单词用下划线连接（如：calculate_sum）
  • 名称要有意义，见名知意
缩进规范：
  • 使用4个空格或Tab键进行缩进
  • 同一层级的代码对齐
  • 大括号单独一行或与语句同行（保持一致）
注释规范：
  • 文件开头说明程序功能、作者、日期
  • 复杂代码段添加注释说明
  • 不要注释显而易见的代码

常见错误及解决方法

编译错误

编译错误是初学者最常遇到的问题，通常是语法不符合C语言规范导致的。以下是几种最常见的编译错误：

常见编译错误：
  • 错误1：缺少分号 — 每条语句末尾必须有分号
  • 错误2：引号不匹配 — 字符串必须使用双引号 "..."
  • 错误3：括号不匹配 — 每个左大括号 { 必须有对应的右大括号 }
  • 错误4：缺少头文件 — 使用 printf 等函数前必须包含 <stdio.h>
  • 错误5：main函数拼写错误 — 注意是 main 而不是 mian

错误代码示例

缺少分号 — 错误 vs 正确
printf("Hello")     //  错误：缺少分号
printf("Hello");    //  正确
引号不匹配 — 错误 vs 正确
printf("Hello');    //  错误：使用了单引号
printf("Hello");    //  正确
缺少头文件 — 错误示例
//  错误：没有包含stdio.h
int main() {
    printf("Hello");    // 编译器不认识printf
    return 0;
}

运行时错误

常见运行时错误：
  • 程序一闪而过：原因：程序执行完毕后窗口立即关闭。解决：使用 Ctrl+F5 运行，或在程序末尾添加 getchar();
  • 输出乱码：原因：字符编码问题，中文显示不正确。解决：确保文件保存为ANSI编码，或在程序中添加 system("chcp 65001");（UTF-8）
