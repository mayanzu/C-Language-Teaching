变量作用域与预处理

理解变量的生命周期和编译前的预处理机制

简介

在编写程序时，我们经常会遇到这样的困惑：为什么有时变量能访问，有时又报错"找不到"？为什么有的变量每次调用函数都会重新初始化，有的却不会？这些都与变量作用域和预处理有关。

本章将帮助你理解：变量在哪些地方"可见"、能活多久，以及程序在编译前会经历怎样的"预处理"阶段。

新手注意：
  • 变量作用域决定了变量的"可见范围"——在哪里能用，在哪里不能用
  • 预处理指令以 # 开头，在编译前由预处理器执行
  • 理解作用域可以避免命名冲突和奇怪的bug
  • 预处理让代码更灵活，可以实现条件编译、代码复用
作用域类比：办公室
  • 全公司公告 = 全局变量（所有地方都能用）
  • 部门通知 = 局部变量（只在部门内有效）
  • 个人笔记 = 块级变量（只有自己能看）
预处理类比：出版前的编辑
  • 写稿 = 写源代码
  • 编辑审稿 = 预处理阶段
  • 删掉"内部资料"字样 = 条件编译
  • 正式出版 = 生成可执行文件

变量作用域

局部变量

局部变量是定义在函数内部的变量，只能在该函数内部访问。就像只有这个部门的人才能看到部门通知。

局部变量的特点：
  • 在函数内部定义
  • 只在定义它的函数内有效
  • 函数调用时创建，函数结束时销毁
  • 每次调用函数都会重新创建和初始化
  • 不同函数中可以定义同名变量，互不影响
局部变量示例
#include <stdio.h>

void funcA() {
    int num = 10;  // funcA的局部变量
    printf("funcA: num = %d\n", num);
}

void funcB() {
    int num = 20;  // funcB的局部变量，和funcA的num是不同的变量
    printf("funcB: num = %d\n", num);
}

int main() {
    int num = 100;  // main函数的局部变量
    funcA();  // 输出: funcA: num = 10
    funcB();  // 输出: funcB: num = 20
    printf("main: num = %d\n", num);  // 输出: main: num = 100
    return 0;
}

全局变量

全局变量是定义在所有函数外部的变量，整个程序都能访问。就像公司公告栏上的通知，所有员工都能看到。

全局变量的特点：
  • 在函数外部定义
  • 从定义处到文件末尾的所有函数都能访问
  • 程序开始时创建，程序结束时销毁
  • 所有函数共享同一个全局变量
  • 可以提供默认值供多个函数共用
全局变量示例
#include <stdio.h>

// 全局变量：在所有函数之外定义
int total = 0;

void add(int value) {
    total += value;  // 可以直接访问全局变量
    printf("加上%d，当前总数: %d\n", value, total);
}

void subtract(int value) {
    total -= value;
    printf("减去%d，当前总数: %d\n", value, total);
}

int main() {
    add(10);     // 输出: 加上10，当前总数: 10
    add(5);      // 输出: 加上5，当前总数: 15
    subtract(3);  // 输出: 减去3，当前总数: 12
    return 0;
}
使用全局变量的注意事项：
  • 全局变量会一直占用内存，直到程序结束
  • 过度使用全局变量会使代码难以维护（数据流向不清晰）
  • 多个文件中使用同名全局变量可能产生冲突，需要用 extern 声明
  • 建议：尽量用参数传递和返回值，少用全局变量

块级作用域

块级作用域指用大括号 {} 包围的代码块。在块内定义的变量只在该块内有效。

块级作用域示例
#include <stdio.h>

int main() {
    int a = 10;

    {
        // 这是另一个代码块
        int b = 20;  // 块级变量
        printf("块内: a = %d, b = %d\n", a, b);  // 可以访问a和b
    }

    printf("块外: a = %d\n", a);  // 可以访问a
    // printf("%d", b);  // 错误！b只在块内有效

    return 0;
}

变量隐藏

当局部变量与全局变量同名时，局部变量会"隐藏"全局变量。在局部作用域内使用的是局部变量，全局变量暂时不可见。

变量隐藏示例
#include <stdio.h>

int value = 100;  // 全局变量

void test() {
    int value = 200;  // 局部变量，隐藏了全局变量
    printf("函数内: value = %d\n", value);  // 200，使用局部变量
}

int main() {
    test();
    printf("函数外: value = %d\n", value);  // 100，使用全局变量

    // 如果想在函数内访问全局变量，可以用 ::value (C++特有)
    // 或者在C语言中，在不同位置定义变量避免同名
    return 0;
}
新手注意：

变量隐藏是合法的，但容易造成混淆。为了代码清晰，建议避免使用与全局变量同名的局部变量。

静态变量

static的作用

static 关键字有两个作用：

  1. 静态局部变量：在函数内部定义，函数调用结束后不销毁，保留其值
  2. 静态全局变量：限制变量的作用域仅在当前文件内，其他文件无法访问
类比：记忆面包

普通局部变量就像"一次性笔记"，函数结束后笔记消失，下次函数调用时重新记。静态局部变量就像"写在黑板上"，函数结束后笔记保留，下次调用还能看到。

静态局部变量示例

静态局部变量 vs 普通局部变量
#include <stdio.h>

void countNormal() {
    int count = 0;  // 普通局部变量，每次调用都重新初始化
    count++;
    printf("普通局部变量: count = %d\n", count);
}

void countStatic() {
    static int count = 0;  // 静态局部变量，只初始化一次
    count++;
    printf("静态局部变量: count = %d\n", count);
}

int main() {
    countNormal();  // count = 1
    countNormal();  // count = 1（重新初始化）
    countNormal();  // count = 1

    countStatic();  // count = 1
    countStatic();  // count = 2（保留上次值）
    countStatic();  // count = 3（继续累加）

    return 0;
}

计数器应用

统计函数调用次数
#include <stdio.h>

void welcome() {
    static int visitCount = 0;
    visitCount++;
    printf("欢迎访问！您是第 %d 位访客。\n", visitCount);
}

int main() {
    welcome();  // 第1位访客
    welcome();  // 第2位访客
    welcome();  // 第3位访客
    return 0;
}

宏定义

#define 宏定义

#define 是最常用的预处理指令，用于定义宏。宏本质上是一段"文本替换"，在编译前，预处理器会把代码中的宏名替换成它代表的值。

宏 vs 变量：
  • 宏在预处理阶段替换，没有数据类型，没有内存地址
  • 变量在程序运行时使用，有数据类型，有内存地址
  • 宏适合定义常量，变量适合存储可能变化的值
宏定义示例
#include <stdio.h>

// 定义常量宏
#define PI 3.14159
#define MAX_SIZE 100
#define YEAR 2024

// 定义带参数的宏（宏函数）
#define MAX(a, b) ((a) > (b) ? (a) : (b))
#define SQUARE(x) ((x) * (x))
#define IS_EVEN(n) ((n) % 2 == 0)

int main() {
    printf("圆周率: %.5f\n", PI);
    printf("最大值: %d\n", MAX(10, 20));
    printf("平方: %d\n", SQUARE(5));

    if (IS_EVEN(8)) {
        printf("8是偶数\n");
    }

    return 0;
}
宏定义注意事项：
  • 宏定义末尾不需要分号，加了反而会替换进去导致语法错误
  • 带参数的宏要加括号避免运算优先级问题：如 #define SQUARE(x) ((x)*(x))
  • 宏是纯文本替换，不进行类型检查
  • 宏没有作用域限制，定义后直到文件末尾都有效

多行宏

如果宏需要多行代码，使用反斜杠 \ 连接。

多行宏示例
#define PRINT_ARRAY(arr, n) \
    for (int i = 0; i < (n); i++) { \
        printf("%d ", (arr)[i]); \
    } \
    printf("\n")

文件包含

#include 指令

#include 指令用于"复制"另一个文件的内容到当前位置。这是代码复用的基础机制，让你可以使用别人写好的函数和声明。

形式 说明 示例
#include <file> 尖括号：查找系统目录 #include <stdio.h>
#include "file" 双引号：先查当前目录，再查系统目录 #include "myheader.h"
使用场景：
  • 系统头文件（stdio.h, stdlib.h 等）用尖括号
  • 自己写的头文件用双引号
  • 头文件通常包含函数声明、宏定义、结构体定义等

头文件编写规范

为了防止头文件被多次包含导致重复定义，通常使用条件编译来保护：

mymath.h - 头文件示例
// mymath.h
#ifndef MYMATH_H
#define MYMATH_H

// 函数声明
int add(int a, int b);
int subtract(int a, int b);

#endif
main.c - 使用头文件
#include <stdio.h>
#include "mymath.h"  // 包含自己写的头文件

int main() {
    printf("3 + 5 = %d\n", add(3, 5));
    return 0;
}

条件编译

条件编译的概念

条件编译允许代码在满足特定条件时才参与编译。这常用于：

  • 同一份代码支持多个平台（Windows/Linux/Mac）
  • 调试代码只在调试模式下编译
  • 根据配置启用/禁用某些功能
类比：选择性出版

就像出版书籍时，根据不同地区选择性地包含某些章节——国内版去掉国外附录，国际版翻译成多语言。条件编译根据"配置"选择性地编译代码。

#if / #else / #endif

条件编译示例
// 根据宏是否定义来选择编译
#define DEBUG 1

int main() {
#if DEBUG == 1
    printf("调试模式：显示详细信息\n");
#else
    printf("发布模式：隐藏调试信息\n");
#endif

    return 0;
}

#ifdef / #ifndef

指令 含义 用途
#ifdef MACRO 如果宏已定义 检查功能开关
#ifndef MACRO 如果宏未定义 防止重复包含
防止重复包含（标准写法）
#ifndef HEADER_FILE_H
#define HEADER_FILE_H

// 头文件内容...

#endif

跨平台开发示例

跨平台代码
#include <stdio.h>

// 模拟不同的操作系统
#define WINDOWS 1
// #define LINUX 1

int main() {
#ifdef WINDOWS
    printf("运行在 Windows 系统上\n");
#elifdef LINUX
    printf("运行在 Linux 系统上\n");
#else
    printf("运行在其他系统上\n");
#endif

    return 0;
}

练习题

就地练习

练习1： 分析以下代码的输出结果：

分析代码
#include <stdio.h>

int x = 1;

void test() {
    int x = 2;
    printf("函数内 x = %d\n", x);
}

int main() {
    test();
    printf("main内 x = %d\n", x);
    return 0;
}
参考答案

输出：

函数内 x = 2

main内 x = 1

原因：函数内的局部变量 x 隐藏了全局变量 x。函数内使用的是局部变量（2），main 中没有同名的局部变量，所以使用的是全局变量（1）。

练习2： 写一个程序，使用静态局部变量统计 greet() 函数被调用的次数。

参考答案
统计函数调用次数
#include <stdio.h>

void greet() {
    static int count = 0;
    count++;
    printf("你好！你是第 %d 次调用。\n", count);
}

int main() {
    greet();
    greet();
    greet();
    return 0;
}

练习3： 使用宏定义计算圆面积（公式：πr²），然后在主函数中使用这个宏。

参考答案
使用宏计算圆面积
#include <stdio.h>

#define PI 3.14159
#define AREA(r) (PI * (r) * (r))

int main() {
    double radius = 5.0;
    printf("半径为 %.1f 的圆面积是 %.2f\n", radius, AREA(radius));
    return 0;
}

练习4： 创建一个简单的头文件 utils.h 和对应的实现，定义一个判断闰年的函数，并在主程序中使用。

参考答案
utils.h
#ifndef UTILS_H
#define UTILS_H

int isLeapYear(int year);

#endif
utils.c
int isLeapYear(int year) {
    return (year % 4 == 0 && year % 100 != 0) || (year % 400 == 0);
}
main.c
#include <stdio.h>
#include "utils.h"

int main() {
    int year = 2024;
    if (isLeapYear(year)) {
        printf("%d 年是闰年\n", year);
    }
    return 0;
}

练习5： 使用条件编译，根据是否定义了 DEBUG 宏来选择性地输出调试信息。

参考答案
条件编译
#include <stdio.h>

// 注释掉这行来关闭调试模式
#define DEBUG 1

int main() {
    int sum = 0;
    for (int i = 1; i <= 5; i++) {
        sum += i;
#ifdef DEBUG
        printf("DEBUG: i = %d, sum = %d\n", i, sum);
#endif
    }
    printf("最终结果: %d\n", sum);
    return 0;
}

练习提示

  • 局部变量在函数结束时销毁，静态变量在整个程序运行期间保留
  • 全局变量可以被多个函数共享，但会增加代码耦合
  • 宏定义没有分号，加括号避免优先级问题
  • 头文件要用条件编译防止重复包含
  • 条件编译可以让同一份代码适配不同环境

本章小结

重点回顾

  • 局部变量：在函数内部定义，函数结束销毁，每次调用重新创建
  • 全局变量：在函数外部定义，整个程序可见，程序结束销毁
  • 块级作用域：在 {} 内定义，只在该块内有效
  • 变量隐藏：局部变量与全局变量同名时，局部变量隐藏全局变量
  • 静态变量：用 static 修饰，函数结束后不销毁，保留值
  • #define：宏定义，文本替换，适合定义常量
  • #include：文件包含，将其他文件内容复制到当前位置
  • 条件编译：#if/#ifdef/#ifndef 根据条件选择编译哪些代码
