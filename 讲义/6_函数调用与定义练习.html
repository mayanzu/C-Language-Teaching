函数调用与定义练习

掌握 C 语言函数的定义、声明和调用方法

函数基础

函数是C语言程序的基本组成单元，用于封装可重复使用的代码块。每个函数都有特定的功能，通过函数名、参数和返回值进行交互。

新手注意：
  • 函数必须先声明后使用
  • main函数是程序的入口点
  • 函数可以调用其他函数

函数定义

函数的组成结构

什么是函数？

函数是C语言中用于封装特定功能代码的基本单元。可以把函数理解为一个"黑盒子"：给它输入（参数），它经过处理后给出输出（返回值）。使用函数可以让代码更加模块化、易于维护和复用。

为什么要使用函数？

  • 代码复用：相同的功能不需要重复编写，一次定义多次使用
  • 模块化：将复杂问题分解成多个小问题，逐个解决
  • 易于维护：修改功能时只需修改对应的函数，不影响其他代码
  • 提高可读性：良好命名的函数让代码意图更清晰

函数的四个组成部分

每个函数都由四个基本部分组成：

  • 返回值类型：函数返回的数据类型（int, double, void等）
  • 函数名：遵循标识符命名规则，建议使用动词或动词短语
  • 参数列表：函数接收的输入数据（可以为空）
  • 函数体：实现功能的代码块，用大括号 {} 包围
函数基本格式
// 函数定义的基本格式
返回值类型 函数名(参数类型1 参数名1, 参数类型2 参数名2) {
    // 函数体：实现功能的代码
    return 返回值;  // void类型不需要return
}

简单示例：求两数之和

有参数有返回值的函数
#include <stdio.h>

// 函数定义：计算两数之和
int add(int a, int b) {
    return a + b;  // 返回结果
}

int main() {
    int sum = add(5, 3);  // 调用函数
    printf("5 + 3 = %d\n", sum);
    return 0;
}
⚠️ 函数定义的注意事项：
  • 函数名应该清晰表达功能（如calculateSum而不是func1）
  • 形参和实参的数量、类型必须匹配
  • return语句会立即结束函数并返回值
  • void函数不能返回值，其他类型函数必须return对应类型的值

函数使用常见错误与注意事项

1. 参数数量不匹配

错误描述：调用函数时提供的实参个数与函数定义的形参个数不一致。

❌ 错误示例
int add(int a, int b) {
    return a + b;
}

int main() {
    int sum = add(5);        // ❌ 错误：缺少第二个参数
    int sum2 = add(5, 3, 2);  // ❌ 错误：参数过多
    return 0;
}

后果：编译错误，程序无法通过编译。编译器会提示"参数数量不匹配"或类似信息。

2. 参数类型不匹配

错误描述：实参的类型与形参的类型不一致，可能导致数据丢失或错误。

⚠️ 类型不匹配示例
void printInt(int n) {
    printf("%d\n", n);
}

int main() {
    double x = 3.14;
    printInt(x);  // ⚠️ 警告：double转int，小数部分丢失，输出3
    
    char *str = "hello";
    // printInt(str);  // ❌ 错误：指针不能直接转int
    
    return 0;
}

后果：可能产生编译警告或错误，运行时可能得到错误结果或数据丢失。

3. 忘记返回值

错误描述：非void函数没有return语句或某些分支没有return。

❌ 忘记返回值
int max(int a, int b) {
    if(a > b) {
        return a;
    }
    // ❌ 错误：当a<=b时没有return语句
}

int getNumber() {
    int n = 10;
    // ❌ 错误：声明返回int但没有return
}

后果：编译警告，运行时返回值不确定（垃圾值），可能导致程序错误。

4. void函数错误使用返回值

错误描述：void函数不应该有返回值，也不能将其用于赋值。

❌ void函数错误
void printHello() {
    printf("Hello\n");
    // return 0;  // ❌ 错误：void函数不能返回值
}

int main() {
    // int x = printHello();  // ❌ 错误：void函数没有返回值
    printHello();  // ✅ 正确：直接调用
    return 0;
}

后果：编译错误，无法通过编译。

5. 函数未声明就使用

错误描述：在函数定义之前调用，且没有提前声明。

❌ 未声明就使用
#include <stdio.h>

int main() {
    int result = add(5, 3);  // ❌ 错误：add函数未声明
    printf("%d\n", result);
    return 0;
}

int add(int a, int b) {
    return a + b;
}
✅ 正确做法
#include <stdio.h>

int add(int, int);  // ✅ 提前声明

int main() {
    int result = add(5, 3);
    printf("%d\n", result);
    return 0;
}

int add(int a, int b) {
    return a + b;
}

后果：C89标准下会产生警告（隐式声明），C99及以后标准会产生编译错误。

6. 参数传递的值传递陷阱

错误描述：误以为值传递可以修改原变量。

❌ 值传递陷阱
void changeValue(int x) {
    x = 100;  // 只修改了副本，不影响原变量
}

int main() {
    int a = 10;
    changeValue(a);
    printf("%d\n", a);  // 输出10，不是100
    return 0;
}
✅ 使用指针修改
void changeValue(int *x) {
    *x = 100;  // 通过指针修改原变量
}

int main() {
    int a = 10;
    changeValue(&a);  // 传递地址
    printf("%d\n", a);  // 输出100
    return 0;
}
📌 最佳实践总结：
  • 参数匹配：确保实参和形参的数量、类型完全对应
  • 函数声明：在使用前声明，或将函数定义放在调用之前
  • 返回值检查：非void函数必须在所有路径上都有return
  • 类型转换：注意隐式类型转换可能导致的精度损失
  • 值传递理解：记住C语言默认是值传递，需要修改原变量时使用指针
  • 编译警告：重视所有编译警告，它们通常指示潜在问题
  • 测试验证：编写函数后要测试各种边界情况

函数声明

函数声明与定义的区别

什么是函数声明？什么是函数定义？

在C语言中，函数的声明和定义是两个不同的概念：

  • 函数声明（Declaration）：告诉编译器函数的名称、返回类型和参数，但不提供函数的具体实现代码
  • 函数定义（Definition）：不仅声明函数的接口，还提供函数的完整实现代码（函数体）
声明与定义对比
#include <stdio.h>

// 函数声明（只有接口，没有实现）
int add(int a, int b);  // 注意：以分号结束

int main() {
    int result = add(5, 3);
    printf("结果: %d\n", result);
    return 0;
}

// 函数定义（有完整的实现代码）
int add(int a, int b) {  // 注意：有大括号和函数体
    return a + b;
}

关键区别总结

特征 函数声明 函数定义
作用 告诉编译器函数的存在 提供函数的完整实现
包含内容 返回类型 + 函数名 + 参数列表 声明 + 函数体（实现代码）
结束符号 分号 ; 大括号 { }
参数名 可以省略（只写类型） 必须写（需要在函数体中使用）
数量限制 可以多次声明（但要一致） 只能定义一次
位置 通常在文件开头或头文件中 可以在任何位置
💡 重要提示：
  • 定义即声明：函数定义自动包含声明，所以如果函数定义在调用之前，可以不写声明
  • 声明不是定义：只有声明没有定义会导致链接错误
  • 多次声明：可以在不同地方多次声明同一个函数，但定义只能有一次

三种使用场景

场景1：函数定义在调用之前（不需要声明）
#include <stdio.h>

// 直接定义函数
int multiply(int a, int b) {
    return a * b;
}

int main() {
    printf("%d\n", multiply(3, 4));  // 可以直接调用
    return 0;
}
场景2：函数定义在调用之后（需要先声明）
#include <stdio.h>

// 先声明
int multiply(int, int);

int main() {
    printf("%d\n", multiply(3, 4));  // 可以调用
    return 0;
}

// 后定义
int multiply(int a, int b) {
    return a * b;
}
场景3：多文件项目（头文件中声明，源文件中定义）
// math_utils.h（头文件 - 包含声明）
#ifndef MATH_UTILS_H
#define MATH_UTILS_H

// 函数声明
int add(int a, int b);
int subtract(int a, int b);

#endif

// math_utils.c（源文件 - 包含定义）
#include "math_utils.h"

// 函数定义
int add(int a, int b) {
    return a + b;
}

int subtract(int a, int b) {
    return a - b;
}

// main.c（主程序 - 使用函数）
#include <stdio.h>
#include "math_utils.h"  // 包含声明

int main() {
    printf("%d\n", add(10, 5));
    printf("%d\n", subtract(10, 5));
    return 0;
}
⚠️ 常见错误：
  • 重复定义：同一个函数在多个文件中定义会导致链接错误
  • 声明不一致：声明和定义的返回类型、参数不匹配会导致编译错误
  • 只声明不定义：只有声明没有定义会导致链接错误
  • 忘记包含头文件：使用函数前忘记包含对应的头文件

函数原型声明

什么是函数原型？

函数原型（Function Prototype）就是函数声明的另一种叫法。它只描述函数的接口（返回值类型、函数名、参数类型），不包含函数的具体实现。

为什么需要函数声明？

  • 编译顺序：C语言从上到下编译，如果函数定义在调用之后，需要先声明
  • 代码组织：可以把函数定义放在文件末尾，保持main函数在前面，便于阅读
  • 多文件开发：在头文件中声明函数，在源文件中定义，实现模块化编程

函数声明的格式

函数声明的基本格式
// 格式：返回值类型 函数名(参数类型列表);
int add(int, int);      // 可以省略参数名
int add(int a, int b);  // 也可以带参数名
void greet(char[]);        // 数组参数

简单示例

函数声明示例
#include <stdio.h>

// 函数声明
int max(int, int);

int main() {
    int a = 10, b = 20;
    printf("较大值: %d\n", max(a, b));
    return 0;
}

// 函数定义在后面
int max(int x, int y) {
    return x > y ? x : y;
}

函数调用

函数调用的方式

如何调用函数？

函数调用就是使用已经定义好的函数。调用时需要提供函数名和必要的参数（实参），函数执行后可能会返回一个值。

函数调用的几种方式

  • 赋值方式：将函数返回值赋给变量
  • 表达式方式：直接在表达式中调用函数
  • 语句方式：单独作为一条语句（常用于void函数）
函数调用示例
#include <stdio.h>

int square(int n) {
    return n * n;
}

int main() {
    // 方式1：赋值给变量
    int result = square(5);
    printf("5的平方: %d\n", result);
    
    // 方式2：直接用在表达式中
    printf("3的平方: %d\n", square(3));
    
    // 方式3：作为参数传递
    printf("结果: %d\n", square(2) + square(3));
    
    return 0;
}

实际应用：简单计算器

#include <stdio.h>

// 函数声明
double add(double, double);
double subtract(double, double);
double multiply(double, double);
double divide(double, double);

int main() {
    double num1, num2, result;
    char op;
    
    printf("简单计算器\n");
    printf("输入表达式 (如: 5 + 3): ");
    scanf("%lf %c %lf", &num1, &op, &num2);
    
    switch(op) {
        case '+':
            result = add(num1, num2);
            break;
        case '-':
            result = subtract(num1, num2);
            break;
        case '*':
            result = multiply(num1, num2);
            break;
        case '/':
            result = divide(num1, num2);
            break;
        default:
            printf("无效运算符\n");
            return 1;
    }
    
    printf("结果: %.2f\n", result);
    return 0;
}

// 函数定义
double add(double a, double b) {
    return a + b;
}

double subtract(double a, double b) {
    return a - b;
}

double multiply(double a, double b) {
    return a * b;
}

double divide(double a, double b) {
    if(b == 0) {
        printf("错误: 除数不能为0\n");
        return 0;
    }
    return a / b;
}

参数传递

形参与实参

什么是形参和实参？

在函数的定义和调用过程中，涉及两种参数：

  • 形参（形式参数）：函数定义时声明的参数，是占位符变量，用于接收传入的值
  • 实参（实际参数）：函数调用时传递的具体值或变量
形参与实参示例
#include <stdio.h>

// 函数定义：a和b是形参
int add(int a, int b) {
    return a + b;
}

int main() {
    int x = 5, y = 3;
    
    // 函数调用：x和y是实参
    int sum1 = add(x, y);     // 变量作为实参
    int sum2 = add(10, 20);  // 常量作为实参
    int sum3 = add(x+1, y*2); // 表达式作为实参
    
    printf("sum1=%d, sum2=%d, sum3=%d\n", sum1, sum2, sum3);
    return 0;
}

形参与实参的关系

  • 数量匹配：实参的个数必须与形参的个数相同
  • 类型匹配：实参的类型应与形参的类型一致（或可以隐式转换）
  • 顺序对应：实参按位置顺序传递给形参
  • 单向传递：实参的值传递给形参，形参是实参的副本
参数匹配示例
#include <stdio.h>

// 形参：type, length, width
void printInfo(char type, int length, int width) {
    printf("类型:%c, 长:%d, 宽:%d\n", type, length, width);
}

int main() {
    // 实参必须按顺序对应形参
    printInfo('A', 10, 5);  // 正确：类型和顺序匹配
    // printInfo(10, 5, 'A');  // 错误：顺序不对
    // printInfo('A', 10);     // 错误：参数个数不足
    return 0;
}
💡 记忆技巧：
  • 形参在函数定义处（形式上的）
  • 实参在函数调用处（实际传入的）
  • 形参相当于"模板"，实参是填入模板的"具体内容"

参数匹配常见问题

参数匹配错误示例
void display(int a, double b, char c) {
    printf("%d %.2f %c\n", a, b, c);
}

int main() {

    // display(10, 3.14);        // 少一个参数 - 编译错误

    // display(3.14, 10, 'A');   // 类型顺序不对 - 可能编译警告

⚠️ 参数传递注意事项：
  • 数量必须完全一致：多一个或少一个都会编译错误
  • 顺序必须对应：第1个实参传给第1个形参，第2个对第2个，依此类推
  • 类型尽量匹配：虽然C语言会自动转换，但可能丢失数据或产生意外结果
  • 表达式作为实参：会先计算表达式的值，再传递给形参

值传递与地址传递

前置知识：指针基础

在学习地址传递之前，我们需要了解指针的基本概念：

  • 指针变量：用来存储变量地址的变量
  • &运算符：取地址运算符，获取变量的内存地址
  • *运算符：解引用运算符，访问指针指向的值
指针基础示例
#include <stdio.h>

    int a = 10;           // 定义整型变量

    printf("a的地址: %p\n", &a);    // 输出地址
    printf("p指向的值: %d\n", *p);  // 输出: 10
    
    *p = 20;           // 通过指针修改a的值
    printf("修改后a: %d\n", a);     // 输出: 20
    
    return 0;
}
💡 理解指针的关键：
  • &a 表示"a的地址"
  • int *p 表示"p是一个指向int的指针"
  • *p 表示"p指向的那个值"
  • 通过指针可以间接访问和修改变量的值

什么是参数传递？

参数传递是指将实参的值传递给函数形参的过程。C语言有两种主要的参数传递方式：值传递和地址传递。

两种传递方式的区别

  • 值传递：传递变量的副本，函数内修改不影响原变量
  • 地址传递：传递变量的地址（指针），函数内可以修改原变量
值传递 vs 地址传递
#include <stdio.h>

// 值传递：不能修改原变量
void modifyValue(int x) {
    x = x * 2;
    printf("函数内x: %d\n", x);
}

// 地址传递：可以修改原变量
void modifyPtr(int *x) {
    *x = *x * 2;
}

int main() {
    int a = 10, b = 10;
    
    modifyValue(a);     // a不变，仍为10
    modifyPtr(&b);      // b改变，变为20
    
    printf("a=%d, b=%d\n", a, b);
    return 0;
}

递归函数

递归的基本概念

什么是递归？

递归是指函数调用自己的编程技巧。递归函数通过将大问题分解成相同类型的小问题来解决，直到问题简单到可以直接求解为止。

递归的两个要素

  • 递归终止条件（基准情况）：必须有明确的结束条件，否则会无限递归
  • 递归调用（递推关系）：函数调用自己，且每次调用都向终止条件靠近
简单递归示例
#include <stdio.h>

// 计算阶乘：n! = n × (n-1)!
int factorial(int n) {
    if (n <= 1) return 1;  // 终止条件
    return n * factorial(n - 1);  // 递归调用
}

int main() {
    printf("5! = %d\n", factorial(5));  // 120
    return 0;
}

递归执行过程详解

以factorial(3)为例，看看递归是如何执行的：

递归执行过程可视化
调用 factorial(3):
  ├─ n = 3, n > 1, 需要计算 3 * factorial(2)
  └─ 调用 factorial(2):
       ├─ n = 2, n > 1, 需要计算 2 * factorial(1)
       └─ 调用 factorial(1):
            ├─ n = 1, n <= 1, 返回 1  ← 到达终止条件
       ├─ 得到 factorial(1) = 1
       └─ 返回 2 * 1 = 2
  ├─ 得到 factorial(2) = 2
  └─ 返回 3 * 2 = 6

最终结果: factorial(3) = 6
💡 理解递归的关键：
  • 递推阶段：函数不断调用自己，问题规模逐渐缩小
  • 回归阶段：达到终止条件后，逐层返回结果并计算
  • 栈的使用：每次调用都在栈上保存当前状态，返回时恢复
  • 效率考虑：递归简洁但可能消耗较多内存，某些情况下循环更高效

递归常见错误

❌ 缺少终止条件导致无限递归
int badFunction(int n) {
    return n + badFunction(n - 1);  // ❌ 没有终止条件！
}
// 结果：栈溢出（Stack Overflow）程序崩溃
❌ 终止条件错误
int factorial(int n) {
    if(n == 1) return 1;  // ❌ 如果传入0或负数会无限递归
    return n * factorial(n - 1);
}

// ✅ 正确的终止条件
int factorial_correct(int n) {
    if(n <= 1) return 1;  // ✅ 考虑了边界情况
    return n * factorial_correct(n - 1);
}
❌ 递归方向错误
int sum(int n) {
    if(n == 10) return 10;
    return n + sum(n + 1);  // ❌ 递增而不是递减，永远达不到10
}

// 应该从大到小递归
int sum_correct(int n) {
    if(n == 1) return 1;
    return n + sum_correct(n - 1);  // ✅ 递减，逐步接近终止条件
}
⚠️ 递归使用注意事项：
  • 必须有明确的终止条件：否则会导致栈溢出（Stack Overflow）
  • 每次递归必须向终止条件靠近：参数要递减或递增到终止值
  • 考虑边界情况：0、负数、空值等特殊输入
  • 注意栈空间限制：递归层数太深（通常几千到上万层）会导致栈溢出
  • 避免重复计算：如斐波那契数列递归效率很低，可考虑循环或记忆化
  • 测试小数据：先用小数据测试，确保逻辑正确后再用大数据
  • 调试技巧：在递归函数中打印参数值，观察递归过程

递归 vs 循环

同一个问题可以用递归或循环实现，各有优劣：

阶乘的两种实现
// 递归实现（简洁但可能效率较低）
int factorial_recursive(int n) {
    if(n <= 1) return 1;
    return n * factorial_recursive(n - 1);
}

// 循环实现（效率高但代码稍长）
int factorial_iterative(int n) {
    int result = 1;
    for (i = 2; i <= n; i++) {
        result *= i;
    }
    return result;
}
